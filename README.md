# psy-platform

Платформа для психологов: мультитул для работы с клиентами, маркетплейс специалистов и инструменты самопомощи. Telegram Mini App + десктопная веб-версия на одном коде.

Полный план — в файле плана (`~/.claude/plans/...`).

## Стек (МВП)

Один Next.js-проект (фулстек): фронт + API в одном коде.

- **Next.js 15 (App Router) + TypeScript** — фронт и бэк.
- **Tailwind CSS** — UI (минималистичный, тёплый монохром).
- **Prisma + PostgreSQL** — данные.
- **jose** — JWT; валидация Telegram `initData` на сервере.
- **ЮKassa** — подписка психолога (единый эквайринг).

Каталог `backend/` (FastAPI) — референс Фазы 0, в разработке не участвует.

## Быстрый старт

Нужны: Docker, bun.

```bash
# 1. База данных
docker compose up -d postgres

# 2. Переменные окружения
cp frontend/.env.example frontend/.env.local
# впиши TELEGRAM_BOT_TOKEN, JWT_SECRET; DATABASE_URL уже под docker

# 3. Приложение
cd frontend
bun install
bun run db:push     # создать таблицы из Prisma-схемы
bun run dev         # http://localhost:3000
```

## Что готово (МВП)

- Вход через Telegram (`initData` → JWT), апсерт психолога.
- Клиентская база: список, карточка с заметками, удаление (проверка владения — без IDOR).
- Расписание: записи (выбор клиента, время, длительность), смена статуса, удаление.
- Напоминания через бота (grammY): за N минут до сессии психологу приходит сообщение.
- Каркас подписки ЮKassa: создание платежа + вебхук (проверка через API, идемпотентность).

## Бот (напоминания)

Отдельный процесс, запускается рядом с приложением:

```bash
cd frontend
bun run bot     # нужен TELEGRAM_BOT_TOKEN и DATABASE_URL
```

Психолог должен один раз нажать `/start` у бота — только после этого Telegram
разрешает боту слать ему сообщения.
