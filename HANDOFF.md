# Продолжение работы на другом устройстве

Короткая инструкция, чтобы сесть за ноутбук (VS Code + Claude Code) и продолжить без потерь.
Общий контекст проекта — в [AGENTS.md](AGENTS.md). Правила для Claude — в [CLAUDE.md](CLAUDE.md) (оба уезжают вместе с репозиторием).

Живой прототип: https://begemot-m.github.io/bereg/

---

## 1. Что нужно на ноутбуке

- **Git**
- **Node.js 22.x** (использовалась 22.14)
- **Bun 1.3+** — основной пакетный менеджер и запуск (`bun run …`)
- **GitHub CLI `gh`** — чтобы следить за деплоем (`gh auth login` при первом запуске)
- **VS Code + расширение Claude Code**
- *(опционально)* Google Chrome — для скриншот-проверки через puppeteer-core

## 2. Развернуть проект

```bash
git clone https://github.com/Begemot-m/bereg.git
cd bereg/frontend
bun install            # postinstall делает prisma generate — БД не нужна
bun run demo           # демо на моках → http://localhost:3000
```

Демо-режиму **не нужен `.env`** — всё живёт в localStorage браузера. `.env` (из `frontend/.env.example`) понадобится только для боевого бэкенда (Prisma/Postgres, ЮKassa, бот).

## 3. Рабочий цикл

```bash
git pull                                            # ВСЕГДА перед началом (репо ведут несколько инструментов)
# …правки…
cd frontend && bunx tsc --noEmit 2>&1 | grep -v "\.next/types"   # проверка типов, должно быть пусто
git add -A && git commit -m "…" && git push origin master        # пуш в master → авто-деплой
```

**Деплой:** пуш в `master` → GitHub Actions собирает статический экспорт демо (`STATIC_EXPORT=1 NEXT_PUBLIC_DEMO=1 PAGES_BASE_PATH=bereg`, CI удаляет `src/app/api`) и публикует на Pages (~2 мин). Проследить:

```bash
gh run watch $(gh run list --limit 1 --json databaseId -q '.[0].databaseId') --exit-status
```

Скриншот-проверка (опционально): puppeteer-core уже в зависимостях; путь к Chrome на ноутбуке будет другой — поправь `executablePath` в разовом скрипте. Мобильный вьюпорт 390×844, DSF 2; скроллить нужно элемент `.overflow-y-auto`, а не window.

## 4. Что уже сделано (последние итерации)

- **Онбординг** переписан в постерном стиле (крупные капсы + слот под картинку). Реальные изображения клади в `frontend/public/onboarding/` с именами `intro-1..2`, `psy-1..3`, `cl-1..4` (`.webp`) — подхватятся автоматически, иначе показывается схематичный экран-заглушка.
- **Каталог**: миниатюры пересобраны «по-человечески» (портрет, «помогаю с…», цитата-маркер, 2 причины, цена+окно, CTA), профиль — вокруг доверия (стиль, почему подходит, первая встреча, голос-демо, «с чем не работает», правила отмены/связи). Убраны звёзды-рейтинги и счётчики.
- **Тарифы**: Вдох PRO **990 ₽** (кабинет: безлимит клиентов, статистика, сводка недели, шаблоны) + Каталог **500 ₽** (размещение, без покупки рейтинга) + Вдох+ 390 ₽. В блоке подписки — таблица сравнения **Free / PRO** и вводная страница сравнения в туториале «Смотреть возможности».
- **Нижнее меню**: закреплено (не тянется в Telegram), прозрачный фон вокруг плашки, активный кружок **плавно скользит** между вкладками (shared `layoutId`), центральная акцентная кнопка.
- **Сессии**: центральный анимированный «крестик» быстрой записи (пружинит, крутится в ×), «График» слева, «Как это работает?» справа.
- **Клиенты**: карточки списка перерисованы (акцент-полоса, крупный аватар с бейджем записи, пилюли метрик).
- **Главная**: карусель разделов объёмнее (декор, крупная иконка, стрелка).
- **Анкета психолога** коррелирует с каталогом: поля `style` / `quote` / `avoids` (маппятся в `Psy`), ручной ввод запросов/языков/стиля, анимированные чипы, экран **«Профиль на проверке»** после публикации (статус → `review`).
- **Терапия**: настроение+статистика над колесом баланса, запись раскрывается прямо в карточке, «Написать» ведёт в Telegram-чат специалиста.

Ключевые файлы: `components/onboarding.tsx`, `app/catalog/page.tsx`, `lib/catalog.ts`, `components/subscription-block.tsx`, `lib/subscription.ts`, `components/app-shell.tsx`, `app/sessions/page.tsx`, `app/clients/page.tsx`, `app/page.tsx`, `components/profile-editor.tsx`, `lib/profile.ts`, `app/therapy/page.tsx`, `lib/demo.ts` (мок-бэкенд).

6. **Боевые платежи ЮKassa, бот-напоминания, IDOR-проверки, безопасность/152-ФЗ** (P0 из аудита).

## 6. Что НЕ переносится через git (учти)

- Твой домашний `~/CLAUDE.md` — **заменён** на `CLAUDE.md` в репозитории (эта же папка), Claude Code на ноуте подхватит его сам.
- Память Claude Code (`~/.claude/projects/.../memory/`) — локальная для машины, для работы не нужна.
- Состояние демо в браузере (localStorage): онбординг и черновики — свои на каждом устройстве/браузере. Онбординг просто покажется заново — это нормально.
- Первый запуск на ноуте: `gh auth login` (для деплой-логов) и, при желании, залогиниться в Claude Code.
