# Публикация анкеты специалиста в каталоге

Дата: 2026-07-20

## Цель

Сделать сохранённый профиль специалиста единственным источником данных для карточки каталога, подробной публичной страницы и предпросмотра в кабинете. Исключить расхождения между заполненной анкетой и тем, что видит клиент.

## Выбранный подход

Единый адаптер преобразует `PsyProfile` в публичную модель `Psy`. Отдельная копия каталожной анкеты не создаётся. Карточка, подробный профиль, фильтры и предпросмотр используют одни и те же поля и справочники.

Отклонены:

- копирование профиля в отдельную запись каталога при сохранении — создаёт риск рассинхронизации;
- немедленный перенос всей модели в новый серверный API — слишком большой объём для текущего прототипа и не нужен для проверки продуктовой логики.

## Условия публикации

Анкета видна клиентам, только если одновременно выполняются условия:

- профиль имеет статус `approved`;
- действует бесплатный пробный период либо активен тариф с размещением в каталоге.

Тариф с инструментами без каталога не публикует анкету. При завершении тарифа или пробного периода анкета полностью исчезает из поиска, подборок и выдачи. Данные профиля сохраняются и автоматически возвращаются в каталог после восстановления доступа.

Если статус подписки не удалось получить, профиль не публикуется. Это исключает случайную публичную выдачу.

Специалист всегда может открыть собственный предпросмотр независимо от статуса проверки и оплаты.

## Единая публичная модель

Адаптер переносит из `PsyProfile`:

- имя и фотографии;
- пол и языки;
- основной и дополнительные методы;
- запросы;
- опыт;
- цену и длительность;
- online/offline/both;
- город, район, метро и точный адрес с учётом `publicExactAddress`;
- описание специалиста;
- описание первой встречи;
- образование;
- Telegram;
- статус проверки.

Системные поля — рейтинг, отзывы, экспозиция, ближайшее окно и время ответа — не заполняются специалистом. Они добавляются каталогом и расписанием.

## Отображение

Карточка показывает крупный основной портрет, имя, проверку, рейтинг, основной метод, опыт, совпадающие запросы, цену, длительность, формат, краткую локацию и ближайшее окно. Точный адрес в карточке не выводится.

Подробный профиль показывает все заполненные поля в порядке принятия решения: условия, причины совпадения, запросы, методы с пояснениями, формат и место, языки, описание, первую встречу, образование, рейтинг и запись. Пустые необязательные разделы скрываются.

Загруженные изображения в формате data URL не получают `basePath`; статические изображения продолжают использовать `/bereg` на GitHub Pages.

## Подписка и честность выдачи

Оплата даёт право на размещение, но не влияет на рейтинг, позицию, причины рекомендации и бейджи качества. Из описания тарифа удаляются обещания «топа выдачи» и «Рекомендуем». Сортировка остаётся основанной на совпадении с запросом клиента, доступности, проверке и честном рейтинге.

## Проверка

Проверяются состояния:

- `review` + trial — только предпросмотр;
- `approved` + trial — опубликовано;
- `approved` + active catalog plan — опубликовано;
- `approved` + tools-only plan — скрыто;
- `approved` + expired/pending/unknown — скрыто;
- заполненные и пустые необязательные поля;
- data URL и статический портрет;
- фильтры, подборка и подробная страница;
- мобильный viewport 390 × 844;
- production build, консоль и GitHub Pages.
