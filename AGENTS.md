# Берег — контекст проекта для AI-агентов

Прочитай это перед работой. Проект ведётся несколькими AI-инструментами по очереди через git.

## Что это
«Берег» — платформа для психологов (Telegram Mini App + веб). Три роли: **гость / клиент / психолог**. Психолог ведёт клиентов, записи, задаёт рабочие часы; клиент записывается в свободные окна. Центр-создатель — «Амур и Психея» (упомянут в кабинете). Живой прототип: https://begemot-m.github.io/bereg/

## Стек
- **Next.js 15 (App Router) + TypeScript**, фронт и бэк в одном проекте (`frontend/`).
- **Tailwind v4**, **motion** (framer-motion) для анимаций, **@phosphor-icons/react** для иконок.
- **Prisma + PostgreSQL** и API-роуты (`src/app/api`) — для боевого режима. НЕ используются в демо.
- **Демо-режим** (`NEXT_PUBLIC_DEMO=1`): всё работает в браузере на моках в `localStorage`, без бэкенда. Именно он задеплоен.

## Как запускать (папка frontend/)
```bash
bun install
bun run demo        # демо на моках → http://localhost:3000  (для разработки/просмотра)
bun run build       # прод-сборка
bunx tsc --noEmit   # проверка типов
```

## Деплой (важно)
Пуш в ветку `master` на GitHub (`Begemot-m/bereg`) → GitHub Actions собирает **статический экспорт** демо и публикует на GitHub Pages. Настроено в `.github/workflows/deploy.yml`.
- Статическая сборка: `STATIC_EXPORT=1 NEXT_PUBLIC_DEMO=1 PAGES_BASE_PATH=bereg`, CI удаляет `src/app/api` (в статике не нужен).
- `next.config.mjs` включает `output: "export"` и `basePath` только когда `STATIC_EXPORT=1`.
- Значит: **любая правка → git push → через ~2 мин на сайте**. Ломать статический экспорт нельзя (нет серверных `redirect()` в страницах, динамические роуты требуют `generateStaticParams`).

## Структура (frontend/src)
- `lib/demo.ts` — мок-бэкенд (перехват `apiFetch`), все данные и правила демо здесь.
- `lib/api.ts` — в демо шлёт в `mockFetch`, иначе на `/api`.
- `lib/role.ts`, `lib/profile.ts` — роль и профиль психолога (localStorage), онбординг-флаг.
- `lib/schedule.ts` / `components/work-hours.tsx` / `components/slot-picker.tsx` — рабочие часы психолога и запись в свободные окна.
- `lib/haptics.ts` — микровибрации (Telegram + navigator.vibrate).
- `components/app-shell.tsx` — навигация (сайдбар/табы), гейт онбординга.
- `components/onboarding.tsx` — знакомство: роль → УТП под роль → анкета психолога.
- `components/ui.tsx`, `components/blocks.tsx` — примитивы (Button/Card/Disclosure/StatTile/ModuleCard).
- `app/page.tsx` (главная, роле-зависимая), `app/sessions`, `app/clients`, `app/cabinet`, `app/catalog`, `app/tools`.
- `components/demo-frame.tsx` — рамка-телефон вокруг демо на десктопе.

## Дизайн-конвенции
- Светлая система: бумага `#f9f8f3`, графит `#2c2e31`, акценты ирис `#5b5bd6` / шалфей `#2e7d63`. Токены в `app/globals.css`.
- **Сплошные заливки** блоков (`.fill-iris/.fill-sage/.fill-ink`), минимум визуального шума, единый радиус (`--r-block`).
- Доп. опции — раскрывать через `<Disclosure>`, не наваливать на экран.
- Иконки только из Phosphor. Без эмодзи в UI.

## Правила
- Отвечать на русском. Комментарии к очевидному коду не добавлять.
- Перед началом — `git pull`; после — `git commit` + `git push` (иначе рассинхрон между инструментами).
- Демо не должно требовать сервера: новые данные — через `lib/demo.ts` (мок), не через реальные `/api` вызовы в демо-режиме.
