# Claude — правила для проекта «Берег»

Перед работой прочитай **[AGENTS.md](AGENTS.md)** (общий контекст проекта) и **[HANDOFF.md](HANDOFF.md)** (как запустить, что сделано, что дальше).

## Язык
Всегда отвечай на русском.

## Стек
- Frontend/Backend в одном Next.js 15 (App Router), TypeScript
- Демо-режим (`NEXT_PUBLIC_DEMO=1`) — моки в localStorage, именно он задеплоен на Pages
- Tailwind v4, motion (framer-motion), @phosphor-icons/react
- Пакетный менеджер — **bun** (не npm/yarn)

## Стиль кода
- TypeScript предпочтительнее JavaScript
- Python (если появится): type hints обязательны, Pydantic для схем
- Не добавляй комментарии к очевидному коду
- Не добавляй docstrings, если не прошу явно
- Не создавай README и документацию без явной просьбы
- Иконки только из Phosphor

## Поведение
- Читай только нужные файлы, не сканируй весь проект без причины
- Не рефактори код за пределами запрошенного изменения
- Не добавляй обработку ошибок для невозможных сценариев
- Не добавляй логирование, если не прошу
- Перед крупными изменениями (>3 файлов) — спроси подтверждение
- **Перед началом работы — `git pull`; после — `git commit` + `git push`** (проект ведётся несколькими инструментами по очереди, иначе рассинхрон)
- git commit/push делай, когда правки готовы и проверены — это ожидаемая часть флоу деплоя (пуш в master → GitHub Pages)

## Стиль письма
- Применяй humanizer ко всем текстовым ответам на русском: живой голос, без канцелярита и шаблонов
- Никаких «таким образом», «следует отметить», «в заключение», «немаловажно»
- Пиши как думающий человек, а не как корпоративный отчёт

## Экономия токенов
- Не повторяй код, который уже есть в контексте
- Показывай только изменённые части файлов
- Для широкой разведки по кодовой базе используй субагентов

## Проверка перед пушем
```bash
cd frontend
bunx tsc --noEmit 2>&1 | grep -v "\.next/types"   # должно быть пусто
```
Демо не должно требовать сервера: новые данные — через `lib/demo.ts` (мок), не через реальные `/api` в демо-режиме. Не ломай статический экспорт (нет серверных `redirect()` в страницах; динамические роуты требуют `generateStaticParams`).
